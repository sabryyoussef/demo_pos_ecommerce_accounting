No system user for cashiers — PIN-only POS (Phase I)

Why this matters in the presentation

In Gate B6, each cashier had an Odoo user (pos.cashier.*) plus an employee record. In Phase I we show the retail pattern many chains prefer: cashiers exist only as hr.employee with a PIN on the till — no mailbox, no backend login, no extra user licence per shift worker. One admin (admin) reviews consolidated sales in the backend.

Database demo_pos_accounting · HTTP port 8025 · Bootstrap: scripts/phaseI_five_cashiers_demo.py · Full runbook: DEMO_SCENARIO_POS_5_EMPLOYEES_ADMIN.md

Live presentation guide (5 shops, 5 cashiers)

Open each POS link in a separate browser tab (or kiosk). On the employee screen, pick the cashier name and enter the PIN. No Odoo user/password is required for the cashier.

  1. Admin (optional opener): Backend login as admin → Commercial Demo dashboard.
  2. Per shop: open the POS UI link below → select employee → enter PIN → sell 2–3 items → payment.
  3. Close: return to backend → Point of Sale → Orders or Sales Details — filter by Employee and Point of Sale.
